03

Definições a serem tomadas

Perguntas para conduzir a conversa com os sócios. A sequência vai do estratégico ao operacional — não pule etapas.

Esta é a pergunta que destrava a conversa. Não comece pelos KPIs — comece pela dor. A resposta vai revelar se o foco é custo, crescimento, retenção ou outro vetor.

→"Se você pudesse resolver uma coisa em pessoas nos próximos 6 meses, o que seria?"
→"O que está custando caro demais e entregando pouco?"
→"Qual decisão de pessoas você tomaria diferente se soubesse o que sabe hoje?"

Ouça sem propor solução. Anote as palavras exatas que os sócios usam — elas vão nomear os KPIs.

Resposta dos sócios: a preencher após a reunião

KPIs sem objetivo de negócio são só números. Quando você sabe para onde a empresa vai, fica óbvio o que medir — e o que ignorar.

→"Vamos dobrar de tamanho? Manter e otimizar? Reestruturar?"
→"Em 12 meses, como seria o time ideal para esse objetivo?"
→"O que RH precisa entregar para que esse objetivo seja possível?"

A resposta determina quais KPIs vão para a primeira linha do dashboard.

Resposta dos sócios: a preencher após a reunião

Muitos gargalos em RH existem porque decisões simples ainda precisam de aprovação dos sócios. Mapear isso define onde o sistema pode ganhar velocidade imediatamente.

→"Quais aprovações de RH chegam pra vocês que poderiam ser automatizadas ou delegadas sem risco?"
→"Onde vocês são consultados hoje mas preferiam ser apenas informados?"
→"O que o sistema precisaria ter para vocês confiarem numa decisão automática?"

Essa conversa define a matriz de autonomia — o coração das decisões críticas do sistema.

Resposta dos sócios: a preencher após a reunião

Essa pergunta revela quais KPIs vão realmente ser usados — e quais vão virar número bonito que ninguém lê. O melhor dashboard é o mínimo que drive decisão.

→"Se você recebesse um relatório de RH toda segunda, quais 3 números você olharia primeiro?"
→"O que te faria agir imediatamente se soubesse?"
→"O que você prefere não saber — e por quê?"

Os 3 números que os sócios citarem são os KPIs primários do sistema. Tudo o mais é secundário.

Decisões críticas do sistema

Estas decisões definem o nível de autonomia do sistema. Sem elas respondidas, o sistema não sabe quando agir sozinho e quando chamar um humano.

01O que o sistema pode fazer sozinho — sem nenhuma intervenção humana?
Opções
AApenas notifica — sempre espera aprovação humana
BAge em transações de baixo risco (férias dentro das regras, reembolso abaixo do limite)
CAge na maioria dos casos, escala apenas anomalias
Impacto no sistema
A = mais controle, mais trabalho manual. C = mais autonomia, requer confiança no sistema. Define quantas interrupções o sistema gera por dia.
Decisão dos sócios: a preencher
02Quando o sistema detecta algo crítico, como e para quem comunica?
Opções
AE-mail para o RH, que decide o que fazer
BSlack / WhatsApp direto para o gestor da área
CNotificação no sistema + resumo semanal para sócios
Impacto no sistema
Define velocidade de resposta e quem carrega a responsabilidade de agir. Alertas no Slack têm resposta mais rápida mas geram mais ruído.
Decisão dos sócios: a preencher
03Com que frequência os sócios recebem o resumo executivo do sistema?
Opções
ASemanal — toda segunda-feira
BQuinzenal
CMensal — todo dia 1
Impacto no sistema
Define o ritmo de governança. Semanal = mais visibilidade, mais tempo de leitura. Mensal = visão estratégica, menos operacional.
Decisão dos sócios: a preencher
04Qual é o orçamento aprovado para construção e manutenção do sistema?
Opções
AOrçamento fixo por fase — aprovação a cada entrega
BOrçamento anual definido com revisão semestral
COrçamento por módulo — cada automação tem seu próprio budget
Impacto no sistema
Define o ritmo de construção e o escopo possível em cada fase. Sem orçamento definido, o roadmap fica sem prioridade real — e decisões de build vs. comprar ficam travadas.
